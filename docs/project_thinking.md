# PaperPilot 项目设计思路与科研工作流说明

> 版本：v1.6.4 | 最后更新：2026-06-08

## 1. 项目背景：科研阅读不只是找论文

科研阅读的困难不只是“能不能搜到论文”。真实场景中，研究者常常会遇到：

- 论文太多，不知道哪些先读；
- 最新论文不一定适合入门；
- 单篇总结不能形成领域视角；
- 初学者需要先了解问题、路线、经典文献和评价方式。

因此，PaperPilot 的目标不是只做一个搜索框或 PDF 总结器，而是帮助用户从论文发现走向知识沉淀。

## 2. 现有工具常见做法与不足

论文搜索平台擅长找论文，但通常需要用户自己判断哪些适合先读。

文献管理工具擅长保存和管理 PDF，但不直接回答“这篇是否值得优先阅读”。

PDF 问答工具能围绕单篇论文提问，但难以帮助用户理解一个领域的整体问题、方法路线和评价方式。

普通 LLM 总结工具可以生成摘要，但往往缺少排序、追踪、问答历史、Reviewer 视角和综述沉淀。

图谱可视化工具能展示论文关系，但如果缺少阅读流程和解释性排序，初学者仍可能不知道从哪里开始。

## 3. PaperPilot 的设计目标

PaperPilot 支持一条完整科研阅读链路：

发现 → 筛选 → 精读 → 批判 → 趋势 → 综述 → 洞察。

在这个流程中，排序不是为了判断论文绝对好坏，而是为了判断：在当前阅读目标下，这篇论文是否值得优先读。

## 4. 为什么需要阅读模式

统一评分模型有天然局限。不同科研阶段的阅读目标不同：

- 前沿追踪需要快速关注新论文、新方法和近期趋势；
- 领域了解需要优先看到综述、代表性方法、benchmark、dataset 和问题定义清楚的论文；
- 深入了解需要筛选适合精读、复现、方法对比和后续研究问题挖掘的候选论文。

v1.6.4 实现的是轻量深入了解模式：仍然只基于 arXiv 标题、摘要和元信息，不接外部文献数据库，不做真实 citation 或 venue 排名。

## 5. 领域了解模式如何建模

领域了解模式关注以下启发式信号：

- 综述/路线价值：survey、review、overview、taxonomy、roadmap 等；
- 经典/代表性信号：发布时间、基础方法/框架/benchmark/dataset 信号、问题-方法-影响表达；
- 权威元信息信号：arXiv journal_ref/comment 中出现会议或期刊关键词；
- 问题定义清晰度：是否清楚说明 problem、challenge、task、objective；
- 方法覆盖度：是否比较、分类、总结多种方法路线；
- benchmark/dataset 价值：是否提供评价任务、数据集、指标、baseline；
- 可读性：摘要长度和结构是否适合快速理解；
- 低权重 freshness：了解领域时不能让最新论文天然占优。

## 6. 深入了解模式为什么存在

领域了解模式解决的是“我怎么进入一个领域”的问题，但科研推进中还会出现另一个阶段：用户已经知道研究方向，希望判断哪些论文值得认真读、复现、对比，或者从哪些局限中发现后续选题。

深入了解模式因此关注：

- 方法完整性：标题和摘要中是否出现框架、架构、算法、训练/推理流程、形式化或实现细节等信号；
- 实验充分性：是否有实验、benchmark、baseline、ablation、analysis、数值结果或理论证明；
- 可复现性信号：是否提到 code、GitHub、implementation details、hyperparameters、dataset release 等线索；
- 权威发表信号：是否在 arXiv 元信息、comment、journal_ref、venue 或 DOI 中出现正式发表线索；
- 方法路线代表性：是否与本次搜索结果中的共同术语重合；
- 局限与讨论信号：是否呈现 limitation、discussion、future work、challenge、open problem 等线索。

这个模式的输出仍然是“候选优先级”，不是论文质量裁判。权威发表是加分项，不是一票否决项；没有 venue 信息的 arXiv-only 论文仍然可以作为候选。

## 7. 局限

- 当前未接 citation count，不能真实判断某篇论文是不是经典论文；
- 当前未接完整 venue 数据库，不能真实判断顶会顶刊权威性；
- 经典性和权威性都是启发式信号，只能作为候选提示；
- 深入了解模式不读取 PDF 全文，不验证代码仓库，不做真实复现难度判断；
- 方法路线代表性信号依赖搜索结果规模和 query 质量，不能代表真实领域主流；
- arXiv 元信息缺失时，权威信号可能不完整；
- 后续可接 OpenAlex、Semantic Scholar、DBLP 增强 citation、venue 和经典文献识别能力。
